# Changelog

## Исправление: Последнее слово не попадало в запись

### Проблема:
При тестировании обнаружилось, что последнее слово вопроса не попадало в аудиофайл.

### Решение:
Добавлена **post-roll задержка** в 500ms после обнаружения паузы, чтобы дать время последним аудио данным записаться в буфер.

### Изменения:

- `apps/desktop/src/App.tsx` - добавлена задержка 500ms перед `get_audio_data()`

### Результат:

✅ Последнее слово теперь попадает в запись  
✅ Полный захват речи от начала до конца

Подробнее: см. `POST_ROLL_FIX.md`

## Улучшение обработки речи - накопление и детекция пауз

### Проблема:
Один вопрос разбивался на несколько частей из-за обработки каждые 5 секунд.

### Решение:
Добавлена логика накопления речи с детекцией пауз.

### Как работает:

1. **Непрерывное накопление** - аудио накапливается в буфере во время речи
2. **Детекция пауз** - система ждет 2 секунды тишины перед обработкой
3. **Обработка целого вопроса** - весь вопрос обрабатывается как единое целое

### Изменения:

- `apps/desktop/src/App.tsx` - добавлена логика накопления речи и детекции пауз
- Используются команды `get_audio_level` и `get_buffer_size` для мониторинга без очистки буфера
- Добавлен визуальный индикатор состояния речи (Говорите / Пауза / Слушаю)

### Параметры:

- **Порог голоса:** 0.02 (2% от максимума, настраивается)
- **Длительность паузы:** 2 секунды
- **Pre-roll буфер:** 3 секунды (захватывает начало речи)
- **Post-roll задержка:** 500ms (захватывает конец речи)

### Преимущества:

✅ Вопросы не разбиваются на части  
✅ Лучшее качество транскрипции (больше контекста)  
✅ Меньше ложных срабатываний  
✅ Естественная обработка речи  
✅ Полный захват речи (начало и конец)

## Улучшение захвата аудио (Pre-roll буфер)

### Проблема:
При тестировании выяснилось, что первое слово интервьюера не попадало в запись.

### Решение:
Добавлен **pre-roll буфер** - кольцевой буфер, который постоянно хранит последние 3 секунды аудио.

### Изменения:

1. **Фоновый поток захвата** - WASAPI поток теперь работает постоянно, даже когда не идет активная запись
2. **Pre-roll буфер** - хранит последние 3 секунды аудио (288KB)
3. **Захват начала речи** - при начале записи pre-roll буфер копируется в основной буфер
4. **Низкая задержка** - первые слова больше не теряются

### Измененные файлы:

- `apps/desktop/src-tauri/src/audio_wasapi.rs` - добавлен pre-roll буфер и фоновый поток

### Технические детали:

- Pre-roll размер: 3 секунды (288,000 байт)
- Формат: 48kHz, mono, 16-bit PCM
- Main buffer: до 30 секунд (2,880,000 байт)

Подробнее: см. `AUDIO_CAPTURE_IMPROVEMENTS.md`

## Автоматическое определение вопросов

### Изменения:

1. **Убрана кнопка "Я отвечаю"** - больше не нужно вручную переключаться между режимами
2. **Автоматическое определение вопросов** - система сама определяет, является ли речь вопросом
3. **Фильтрация мусора** - игнорируются приветствия, шум, короткие фразы

### Новые файлы:

- `apps/api/src/routes/detect-question.ts` - эндпоинт для определения вопросов
- `apps/api/src/routes/transcribe.ts` - отдельный роут для транскрипции
- `apps/api/src/routes/generate.ts` - отдельный роут для генерации ответов

### Измененные файлы:

- `apps/api/src/services/openai-service.ts` - добавлен метод `detectQuestion()`
- `apps/desktop/src/services/interview-service.ts` - добавлен метод `detectQuestion()`
- `apps/desktop/src/app.tsx` - убрана кнопка "Я отвечаю", добавлена автоматическая проверка
- `apps/desktop/src/store/app-store.ts` - добавлен тип "system" для сообщений
- `apps/desktop/src/components/chat.tsx` - поддержка системных сообщений
- `apps/desktop/src/components/chat.css` - стили для системных сообщений
- `packages/types/src/api.ts` - добавлен тип `DetectQuestionResponse`

### Как это работает:

1. Система захватывает аудио каждые 5 секунд
2. Проверяет уровень громкости (фильтрует тишину)
3. Транскрибирует речь через Whisper API
4. Отправляет транскрипцию в GPT-4o-mini для определения, является ли это вопросом
5. Если это вопрос - генерирует ответ через GPT-4o
6. Если нет - показывает системное сообщение и пропускает

### Примеры:

**Определяется как вопрос:**
- "Что такое React hooks?"
- "Расскажите о вашем опыте"
- "Как бы вы решили эту задачу?"

**НЕ определяется как вопрос:**
- "Привет"
- "Хорошо, понятно"
- "Спасибо"
- Фоновый шум
