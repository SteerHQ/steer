# Voice Activity Detection (VAD)

## Проблема

Система захватывает аудио каждые 5 секунд, но не всегда в этом аудио есть речь:
- Тишина между вопросами
- Фоновый шум
- Музыка/звуки из Zoom
- Пустой эфир

Отправка пустого аудио в API:
- ❌ Тратит деньги (Whisper API платный)
- ❌ Замедляет работу (лишние запросы)
- ❌ Создает мусор в истории

## Решение: Voice Activity Detection

Перед отправкой в API проверяем уровень звука в аудио:
- Если уровень **выше порога** → есть речь → отправляем
- Если уровень **ниже порога** → тишина/шум → пропускаем

## Как это работает

### 1. Расчет уровня звука (RMS)

```typescript
function calculateAudioLevel(buffer: number[]): number {
  // PCM данные - 16-bit signed integers
  const sampleCount = Math.floor(buffer.length / 2);
  let sum = 0;

  for (let i = 0; i < sampleCount; i++) {
    // Конвертируем 2 байта в signed 16-bit sample
    const sample = (buffer[i*2] | (buffer[i*2+1] << 8));
    const signedSample = sample > 32767 ? sample - 65536 : sample;
    
    // Нормализуем к -1.0 ... 1.0
    const normalized = signedSample / 32768.0;
    
    // Суммируем квадраты
    sum += normalized * normalized;
  }

  // RMS (Root Mean Square)
  return Math.sqrt(sum / sampleCount);
}
```

**Результат:** число от 0.0 (тишина) до 1.0 (максимальная громкость)

### 2. Сравнение с порогом

```typescript
const audioLevel = calculateAudioLevel(audioBuffer);
const VOICE_THRESHOLD = 0.02; // 2%

if (audioLevel < VOICE_THRESHOLD) {
  console.log('Silence/noise, skipping');
  return; // Не отправляем в API
}

// Уровень достаточный, обрабатываем
transcribeAudio(audioBuffer);
```

Порог можно настроить через UI компонент `VoiceSensitivity`:

```
[████████░░░░░░░░░░] 3.5%

```

## Рекомендуемые пороги

| Порог | Описание | Когда использовать |
|-------|----------|-------------------|
| 0.5% | Очень низкий | Тихий микрофон, шепот |
| 1.0% | Низкий | Чувствительный режим |
| **2.0%** | **Средний** | **Рекомендуется для большинства** |
| 5.0% | Высокий | Шумная среда |
| 10.0% | Очень высокий | Только громкая речь |

## Настройка

### Если система не реагирует на вопросы

**Проблема:** Порог слишком высокий

**Решение:**
1. Откройте режим Interview
2. Найдите компонент "🎚️ Чувствительность голоса"
3. Сдвиньте слайдер **влево** (уменьшите порог)
4. Попробуйте задать вопрос снова

### Если система обрабатывает тишину

**Проблема:** Порог слишком низкий

**Решение:**
1. Откройте режим Interview
2. Найдите компонент "🎚️ Чувствительность голоса"
3. Сдвиньте слайдер **вправо** (увеличьте порог)
4. Дождитесь тишины - система не должна реагировать

## Технические детали

### Почему RMS?

RMS (Root Mean Square) - стандартный метод измерения уровня звука:
- Учитывает амплитуду всех сэмплов
- Не зависит от знака (положительные/отрицательные сэмплы)
- Дает реалистичную оценку громкости

### Альтернативы

**Peak detection** (максимальная амплитуда):
```typescript
const peak = Math.max(...samples.map(Math.abs));
```
❌ Проблема: один громкий сэмпл = ложное срабатывание

**Zero-crossing rate** (частота пересечения нуля):
```typescript
const zcr = countZeroCrossings(samples);
```
❌ Проблема: не различает речь и шум

**Spectral analysis** (частотный анализ):
```typescript
const fft = performFFT(samples);
const voiceFrequencies = fft.slice(300, 3400); // 300-3400 Hz
```
✅ Лучше, но сложнее и медленнее

## Заключение

Voice Activity Detection экономит деньги и ускоряет работу, отфильтровывая тишину и шум перед отправкой в API. Порог можно настроить через UI для оптимальной работы в разных условиях.
