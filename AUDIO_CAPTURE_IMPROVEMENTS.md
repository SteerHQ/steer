# Улучшения захвата аудио

## Проблема
При тестировании выяснилось, что первое слово интервьюера не попадало в запись, хотя все записывалось.

## Причина
Классическая проблема с Voice Activity Detection (VAD) - детектор срабатывает с задержкой, и начало речи теряется.

## Решение

### 1. Pre-roll буфер (кольцевой буфер)
Добавлен постоянно работающий кольцевой буфер, который хранит последние **3 секунды** аудио (288,000 байт при 48kHz mono 16-bit).

**Как работает:**
- Фоновый поток WASAPI постоянно захватывает системное аудио
- Все аудио записывается в кольцевой pre-roll буфер
- При вызове `start_capture()` содержимое pre-roll буфера копируется в основной буфер
- Это гарантирует, что начало речи не будет потеряно

### 2. Архитектура

```
┌─────────────────────────────────────────┐
│   WASAPI Background Thread              │
│   (всегда работает)                     │
└─────────────────┬───────────────────────┘
                  │
                  ├──► Pre-roll Buffer (3 сек)
                  │    [кольцевой буфер]
                  │
                  └──► Main Buffer (при is_capturing=true)
                       [до 30 сек]
```

### 3. Изменения в коде

**apps/desktop/src-tauri/src/audio_wasapi.rs:**
- Добавлен `pre_roll_buffer: Arc<Mutex<VecDeque<u8>>>`
- Добавлен `is_thread_running` для управления фоновым потоком
- Фоновый поток запускается при создании `WasapiCapture::new()`
- `start_capture()` теперь только копирует pre-roll в main buffer
- `stop_capture()` останавливает запись, но pre-roll продолжает работать

### 4. Преимущества

✅ **Захват начала речи** - первые слова не теряются  
✅ **Низкая задержка** - pre-roll буфер всегда готов  
✅ **Минимальное потребление памяти** - только 288KB для pre-roll  
✅ **Прозрачность** - изменения не требуют модификации TypeScript кода

### 5. Параметры

- **Pre-roll размер:** 3 секунды (288,000 байт)
- **Формат:** 48kHz, mono, 16-bit PCM
- **Main buffer:** до 30 секунд (2,880,000 байт)

### 6. Тестирование

Для проверки работы:
1. Запустите приложение
2. Начните говорить сразу после начала записи
3. Первое слово должно быть захвачено полностью

### 7. Дальнейшие улучшения (опционально)

Если потребуется, можно добавить:
- Настраиваемый размер pre-roll буфера
- Post-roll (продолжение записи после окончания речи)
- Адаптивный VAD с учетом фонового шума
