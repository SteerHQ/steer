# Interview Mode Design

## Концепция

Режим интервью для разработчика:
- **Системное аудио (WASAPI Loopback)** - захватывает **вопросы интервьюера** из Zoom/Teams/Meet
- **Ответы кандидата НЕ фиксируются** - только вопросы для анализа и подготовки

## Архитектура

### Основной подход: Непрерывный захват системного аудио

```
┌─────────────────────────────────────────────────────┐
│  Постоянно: Системное аудио (Loopback)              │
│  → Вопросы интервьюера из Zoom/Teams/Meet           │
│  → Транскрипция → Анализ → Подсказки для ответа     │
└─────────────────────────────────────────────────────┘
```

**Проблема:** Как отличить вопрос интервьюера от ответа кандидата?

### Решение 1: Маркер "Я говорю" (РЕКОМЕНДУЕТСЯ)
**Простой и надежный**

Кандидат нажимает кнопку "Я отвечаю" когда начинает говорить:
- Когда кнопка НЕ нажата → записываем и анализируем (вопрос интервьюера)
- Когда кнопка нажата → игнорируем (ответ кандидата)

**UI:**
```
┌──────────────────────────────────────┐
│  🔴 Слушаю вопросы                   │
│  [Нажмите когда отвечаете] 🎤        │
│                                      │
│  Последний вопрос:                   │
│  "Расскажите о вашем опыте с React"  │
└──────────────────────────────────────┘

Когда кандидат нажимает:
┌──────────────────────────────────────┐
│  ⏸️  Пауза (вы отвечаете)            │
│  [Отпустите когда закончите] 🎤      │
│                                      │
│  Подсказка для ответа:               │
│  "Упомяните hooks, context, redux"   │
└──────────────────────────────────────┘
```

### Решение 2: Voice Activity Detection (VAD)
**Автоматический, экспериментальный**

Определение говорящего по характеристикам голоса:
- Анализ частоты и тембра
- Машинное обучение для разделения голосов
- Требует калибровки

**Проблемы:**
- Сложная реализация
- Требует обучения модели
- Может ошибаться

## Реализация "Я говорю" (Рекомендуемая)

### 1. Frontend (React)

Новый компонент `SpeakingToggle.tsx`:

```typescript
export const SpeakingToggle = () => {
  const [isSpeaking, setIsSpeaking] = useState(false);
  
  const handleToggle = () => {
    setIsSpeaking(!isSpeaking);
    // Когда кандидат говорит - приостанавливаем обработку
    if (!isSpeaking) {
      pauseAudioProcessing();
    } else {
      resumeAudioProcessing();
    }
  };
  
  return (
    <button 
      onClick={handleToggle}
      className={isSpeaking ? 'speaking' : 'listening'}
    >
      {isSpeaking ? '⏸️ Я отвечаю' : '🔴 Слушаю вопросы'}
    </button>
  );
};
```

### 2. Логика обработки

```typescript
const processAudioPipeline = async () => {
  // Пропускаем обработку если кандидат говорит
  if (isCandidateSpeaking) {
    return; // Игнорируем аудио
  }
  
  // Обрабатываем только когда кандидат НЕ говорит
  // (т.е. говорит интервьюер)
  const audioBuffer = await invoke('get_audio_data');
  const transcript = await transcribeAudio(audioBuffer);
  
  // Это вопрос интервьюера
  addInterviewerQuestion(transcript);
  
  // Генерируем подсказку для ответа
  const suggestion = await generateAnswerSuggestion(transcript);
  showSuggestion(suggestion);
};
```

### 3. Workflow

```
1. Системное аудио постоянно захватывается
   ↓
2. Интервьюер задает вопрос
   ↓
3. Аудио обрабатывается → транскрипция
   ↓
4. Вопрос анализируется → генерируется подсказка
   ↓
5. Кандидат нажимает "Я отвечаю"
   ↓
6. Обработка приостанавливается
   ↓
7. Кандидат отвечает (аудио игнорируется)
   ↓
8. Кандидат отпускает кнопку
   ↓
9. Система снова слушает вопросы
```

## API Changes

### Новый endpoint для вопросов интервьюера

```typescript
POST /api/transcribe-question
Content-Type: audio/wav

Response:
{
  "success": true,
  "question": "Расскажите о вашем опыте с React",
  "addedToContext": true
}
```

### Обновленный контекст

```typescript
interface InterviewContext {
  questions: Array<{
    text: string;
    timestamp: number;
    source: 'interviewer' | 'candidate';
  }>;
  answers: Array<{
    text: string;
    timestamp: number;
    relatedQuestion?: string;
  }>;
}
```

## Приоритет реализации

1. ✅ **Phase 1**: PTT кнопка + базовая запись
2. ✅ **Phase 2**: Интеграция с API + контекст
3. ⏳ **Phase 3**: Горячие клавиши (SPACE)
4. ⏳ **Phase 4**: Визуальная индикация
5. 🔮 **Future**: VAD (опционально)

## Альтернативный подход: Два окна

Если PTT неудобен, можно сделать два независимых окна:
- **Главное окно**: Захват системного аудио (ответы)
- **Popup окно**: Кнопка "Задать вопрос" → запись → отправка

Это проще в реализации и понятнее для пользователя.
