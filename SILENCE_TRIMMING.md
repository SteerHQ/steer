# Обрезка тишины из аудио файлов

## Проблема

В аудио файл попадала пустота (тишина) до и после речи, что:
- Увеличивало размер файла
- Замедляло обработку Whisper API
- Ухудшало качество транскрипции
- Тратило токены на обработку тишины

## Решение

Добавлена автоматическая обрезка тишины с сохранением небольших отступов для корректного захвата начала и конца речи.

## Как работает

### Алгоритм обрезки

```rust
fn trim_silence(buffer: &[u8], threshold: f32) -> Vec<u8> {
    // 1. Находим первый не-тихий сэмпл
    for sample in samples {
        if abs(sample) > threshold {
            start_idx = i;
            break;
        }
    }
    
    // 2. Находим последний не-тихий сэмпл
    for sample in samples.reverse() {
        if abs(sample) > threshold {
            end_idx = i;
            break;
        }
    }
    
    // 3. Добавляем padding (0.2 сек до и после)
    start_with_padding = start_idx - padding;
    end_with_padding = end_idx + padding;
    
    // 4. Возвращаем обрезанный буфер
    return buffer[start_with_padding..end_with_padding];
}
```

### Параметры

| Параметр | Значение | Описание |
|----------|----------|----------|
| Порог тишины | 0.02 (2%) | Уровень ниже которого считается тишиной |
| Padding | 0.2 секунды | Отступ до и после речи |
| Формат | 16-bit PCM | Моно, 48kHz |

### Визуализация

```
До обрезки:
[тишина 2 сек][речь 5 сек][тишина 3 сек]
├──────────────┼───────────┼──────────────┤
0              2           7              10 сек

После обрезки:
[padding 0.2][речь 5 сек][padding 0.2]
├────────────┼───────────┼────────────┤
0            0.2         5.2          5.4 сек

Экономия: 10 сек → 5.4 сек (46% меньше!)
```

## Преимущества

### 1. Меньший размер файла

**Пример:**
```
До:  10 секунд × 48kHz × 2 bytes = 960,000 bytes
После: 5.4 секунд × 48kHz × 2 bytes = 518,400 bytes
Экономия: 441,600 bytes (46%)
```

### 2. Быстрее обработка

**Whisper API:**
- Меньше данных → быстрее обработка
- Меньше токенов → дешевле
- Лучше качество транскрипции (нет тишины)

### 3. Точная транскрипция

Whisper API лучше работает с чистой речью без длинных пауз.

### 4. Правильные границы

Padding 0.2 секунды гарантирует:
- ✅ Начало речи не обрезано
- ✅ Конец речи не обрезан
- ✅ Естественные переходы

## Технические детали

### Определение тишины

```rust
let threshold_i16 = (threshold * 32768.0) as i16;

for sample in samples {
    if abs(sample) > threshold_i16 {
        // Это НЕ тишина
    } else {
        // Это тишина
    }
}
```

**Порог 0.02 (2%):**
- 0.02 × 32768 = 655
- Сэмплы с амплитудой < 655 считаются тишиной

### Padding

```rust
let padding_samples = (48000.0 * 0.2) as usize; // 9600 сэмплов
let start_with_padding = start_idx.saturating_sub(padding_samples);
let end_with_padding = (end_idx + padding_samples).min(sample_count);
```

**0.2 секунды:**
- 48kHz × 0.2 сек = 9600 сэмплов
- 9600 × 2 bytes = 19,200 bytes padding

## Настройка

### Изменить порог тишины

В `commands.rs`:
```rust
let trimmed_buffer = trim_silence(&buffer, 0.03); // 3% вместо 2%
```

**Когда увеличить:**
- Много фонового шума
- Ложные срабатывания на шум

**Когда уменьшить:**
- Очень тихая речь
- Обрезается начало/конец

### Изменить padding

В `commands.rs`:
```rust
let padding_samples = (48000.0 * 0.3) as usize; // 0.3 сек вместо 0.2
```

**Когда увеличить:**
- Обрезается начало/конец речи
- Нужны более плавные переходы

**Когда уменьшить:**
- Нужен минимальный размер файла
- Речь очень четкая
