# Как использовать режим интервью

## Что это?

AI-ассистент для прохождения технических собеседований на позицию Software Engineer:
- Захватывает **вопросы интервьюера** из Zoom/Teams/Meet
- Генерирует **готовые ответы**, которые можно зачитать
- **НЕ записывает** вашу речь когда вы отвечаете

## Как это работает?

1. Интервьюер задает вопрос в Zoom/Teams/Meet
2. Система захватывает и транскрибирует вопрос
3. AI генерирует готовый ответ (3-5 предложений)
4. Вы зачитываете этот ответ интервьюеру
5. Зажимаете SPACE чтобы система не обрабатывала вашу речь
6. Отпускаете SPACE → система слушает следующий вопрос

## Как работает?

### 1. Включите режим интервью

В главном окне выберите режим **"Interview"**

### 2. Используйте кнопку "Я отвечаю"

Когда вы видите кнопку:

**🔴 Слушаю вопросы** (зеленая)
- Система активна
- Захватывает и анализирует вопросы интервьюера
- Вы НЕ говорите

**⏸️ Я отвечаю... 5.2s** (желтая)
- Обработка приостановлена
- Вы отвечаете на вопрос
- Ваша речь НЕ записывается

### 3. Горячие клавиши

**SPACE** - зажмите когда отвечаете, отпустите когда закончите

Это быстрее чем кликать мышкой!

## Workflow

```
1. Интервьюер задает вопрос
   ↓
2. Система транскрибирует вопрос
   ↓
3. Генерируется подсказка для ответа
   ↓
4. Вы видите подсказку в чате
   ↓
5. Зажимаете SPACE (или кликаете кнопку)
   ↓
6. Отвечаете на вопрос
   ↓
7. Отпускаете SPACE
   ↓
8. Система снова слушает вопросы
```

## Пример использования

### Интервьюер спрашивает:
> "Расскажите о вашем опыте с React"

### Система показывает:
```
📝 Вопрос интервьюера:
"Расскажите о вашем опыте с React"

🤖 Готовый ответ (зачитайте):
"Я работаю с React уже более 3 лет и активно использую его 
в production проектах. Особенно мне нравится работать с hooks - 
useState для локального состояния, useEffect для побочных эффектов, 
и useContext для глобального состояния. В последнем проекте я 
разработал систему компонентов с использованием TypeScript и 
React Query для управления серверным состоянием, что значительно 
упростило работу с API."
```

### Вы:
1. Читаете ответ (можно адаптировать под себя)
2. Зажимаете SPACE
3. Зачитываете ответ интервьюеру
4. Отпускаете SPACE

### Система:
Снова слушает следующий вопрос

## Настройки

### Включить/выключить анализ

Кнопка **"🤖 Анализ ВКЛ"** / **"⏸️ Анализ ВЫКЛ"**

Когда анализ выключен:
- Аудио захватывается, но не отправляется в API
- Экономит API запросы
- Полезно для пауз в интервью

### Выбор аудиоустройства

В настройках (⚙️) выберите:
- **WASAPI Loopback** - для захвата системного аудио (рекомендуется)
- Или конкретное устройство

## Советы

✅ **Зажимайте SPACE сразу** когда начинаете отвечать
✅ **Не отпускайте** пока не закончите говорить
✅ **Используйте подсказки** как план ответа, не читайте дословно
✅ **Проверьте аудиоустройство** перед интервью

❌ Не забывайте отпускать SPACE после ответа
❌ Не включайте анализ если не нужен (экономия API)

## Troubleshooting

### Система не слышит вопросы
- Проверьте что выбрано правильное аудиоустройство
- Убедитесь что WASAPI Loopback работает
- Проверьте что анализ включен (🤖 Анализ ВКЛ)

### Система обрабатывает мои ответы
- Убедитесь что зажимаете SPACE когда отвечаете
- Проверьте что кнопка показывает "⏸️ Я отвечаю"

### Подсказки не появляются
- Проверьте что API сервер запущен (localhost:3000)
- Проверьте что OPENAI_API_KEY настроен в apps/api/.env
- Посмотрите логи в консоли

## Приватность

⚠️ **Важно:**
- Вопросы интервьюера отправляются в OpenAI API
- Ваши ответы **НЕ записываются** и **НЕ отправляются**
- Все данные обрабатываются локально через ваш API ключ
- История хранится только локально в приложении
