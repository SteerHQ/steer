# Руководство по использованию Voice Assistant

**🎯 Специально оптимизирован для помощи на технических собеседованиях!**

## Быстрый старт для Google Meet на Windows

### 1. Установка VB-Cable

VB-Cable - это виртуальный аудио кабель, который позволяет перехватывать системный звук.

1. Скачайте VB-Cable с официального сайта: <https://vb-audio.com/Cable/>
2. Распакуйте архив
3. Запустите `VBCABLE_Setup_x64.exe` от имени администратора
4. Перезагрузите компьютер после установки

### 2. Настройка Windows Audio

#### Настройка устройства воспроизведения

1. Откройте **Параметры Windows** → **Система** → **Звук**
2. В разделе **Вывод** выберите **CABLE Input (VB-Audio Virtual Cable)** как устройство по умолчанию
3. Теперь весь системный звук будет идти через VB-Cable

#### Настройка прослушивания (чтобы слышать звук)

1. Откройте **Панель управления** → **Звук**
2. Перейдите на вкладку **Запись**
3. Найдите **CABLE Output**, нажмите правой кнопкой → **Свойства**
4. Перейдите на вкладку **Прослушать**
5. Поставьте галочку **Прослушивать с данного устройства**
6. Выберите ваши реальные динамики/наушники в выпадающем списке
7. Нажмите **Применить** и **OK**

### 3. Настройка Voice Assistant

1. Запустите приложение **VoiceAssistant**
2. При первом запуске откроется окно настроек:
   - **OpenAI API ключ**: Введите ваш API ключ от OpenAI (получить можно на <https://platform.openai.com/api-keys>)
   - **Аудиоустройство**: Выберите **CABLE Output (Output/Loopback)** из списка
   - **Позиция overlay**: Выберите где будут появляться уведомления
   - **Автоскрытие**: Настройте через сколько секунд скрывать уведомления
   - **Режим по умолчанию**: Выберите режим работы (рекомендуется "Собеседование")
3. Нажмите **Сохранить**

### 3.1. Режимы работы

Voice Assistant поддерживает 4 специализированных режима:

- **🎯 Собеседование** - Краткие технические ответы с контекстом предыдущих вопросов (рекомендуется для интервью)
- **🧮 Алгоритмы** - Помощь с алгоритмическими задачами, сложность O(n), псевдокод
- **📝 Шпаргалка** - Быстрые факты, определения, синтаксис без объяснений
- **💬 Общий** - Стандартный режим для обычных вопросов

**Переключение режимов:** Кликните на текущий режим в главном окне и выберите нужный.

**Подробнее:** См. [Документацию по режимам](docs/interview-modes.md) и [Примеры использования](docs/examples-ru.md)

### 4. Использование в Google Meet

#### Подготовка

1. Убедитесь, что VB-Cable настроен (см. шаг 2)
2. Запустите Voice Assistant
3. Дождитесь появления визуализатора звука (полоски должны реагировать на звук)

#### Во время встречи

1. Присоединитесь к Google Meet
2. В настройках Google Meet выберите ваш обычный микрофон (не VB-Cable!)
3. Voice Assistant автоматически:
   - Захватывает звук из Google Meet через VB-Cable
   - Распознаёт речь через Whisper API
   - Генерирует краткие ответы через GPT-4
   - Показывает результаты в чате и overlay-уведомлениях

#### Что вы увидите

- **Визуализатор звука**: Полоски анимируются когда кто-то говорит
- **Чат**: История всех распознанных фраз и ответов ассистента
- **Overlay**: Всплывающие уведомления с ответами (справа внизу экрана)

### 5. Управление приложением

#### Кнопки управления

- **Выбор режима** - Кликните на текущий режим (🎯/🧮/📝/💬) для переключения
- **💾 Сохранить аудио** - Сохраняет текущий аудио буфер в файл для отладки
- **🗑️ Очистить чат** - Удаляет всю историю сообщений
- **⚙️ Настройки** - Открывает окно настроек

#### Управление контекстом (режим собеседования)

В режиме собеседования ассистент запоминает последние 10 вопросов и ответов:

- **Просмотр истории** - Кликните на режим → увидите количество вопросов в истории
- **Очистка истории** - Кликните на режим → "🗑️ Очистить историю (N)"
- **Автоочистка** - История автоматически очищается при переключении режима

#### Заголовок окна

- **Перетаскивание** - Зажмите левую часть заголовка для перемещения окна
- **−** - Свернуть окно
- **□** - Развернуть/восстановить окно
- **×** - Закрыть приложение

### 6. Проверка работы

#### Тест аудио захвата

1. Включите любую музыку или видео на компьютере
2. Проверьте визуализатор - полоски должны двигаться
3. Если полоски не двигаются:
   - Проверьте что CABLE Input выбран как устройство воспроизведения по умолчанию
   - Проверьте что в настройках приложения выбран CABLE Output

#### Тест распознавания

1. Нажмите кнопку **💾 Сохранить аудио**
2. Откройте сохранённый WAV файл (путь покажется в уведомлении)
3. Проверьте что файл содержит звук
4. Файлы сохраняются в: `Документы\VoiceAssistant\debug\`

### 7. Решение проблем

#### Не слышу звук из Google Meet

- Проверьте настройку прослушивания VB-Cable (шаг 2)
- Убедитесь что выбраны правильные динамики в настройках прослушивания

#### Визуализатор не показывает активность

- Проверьте что CABLE Output выбран в настройках приложения
- Проверьте что CABLE Input установлен как устройство воспроизведения по умолчанию
- Перезапустите приложение

#### Ошибка "Device not found"

- Убедитесь что VB-Cable установлен
- Перезагрузите компьютер после установки VB-Cable
- Проверьте что устройство видно в Windows Sound Settings

#### Плохое качество распознавания

- Проверьте качество аудио через кнопку "Сохранить аудио"
- Убедитесь что громкость системы достаточная
- Проверьте что микрофон в Google Meet работает нормально

### 8. Важные замечания

⚠️ **Внимание:**

- Приложение обрабатывает аудио каждые 5 секунд
- Для работы требуется активное интернет-соединение (OpenAI API)
- API ключ хранится локально в браузере
- История чата не сохраняется между сеансами

💡 **Советы:**

- Держите окно приложения поверх других окон (alwaysOnTop включен по умолчанию)
- Используйте overlay-уведомления для быстрого просмотра ответов
- Очищайте чат периодически для лучшей производительности
- Сохраняйте аудио для отладки если что-то работает неправильно

### 9. Системные требования

- **ОС**: Windows 10/11
- **Интернет**: Стабильное подключение для OpenAI API
- **Аудио**: VB-Cable установлен и настроен
- **API**: Действующий OpenAI API ключ с доступом к Whisper и GPT-4

### 10. Конфиденциальность

- Аудио обрабатывается через OpenAI API
- Данные не сохраняются на серверах приложения
- API ключ хранится только локально
- История чата хранится только в памяти приложения
