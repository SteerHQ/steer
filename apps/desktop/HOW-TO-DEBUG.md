# Как отладить запись аудио

## 🚀 Быстрый старт

### 1. Запустите приложение
```powershell
cd apps/desktop
bun run tauri:dev
```

### 2. Откройте компонент отладки
Нажмите кнопку **"🔧 Отладка"** внизу экрана

### 3. Выберите устройство
- В списке найдите **VB-Cable**
- Для захвата системного звука выбирайте устройство с **(Output/Loopback)**

### 4. Начните захват
- Нажмите **"▶️ Начать захват"**
- Статус изменится на **"🔴 Запись"**

### 5. Воспроизведите звук
- Включите музыку, видео или разговор
- Убедитесь, что звук идет через VB-Cable
- Наблюдайте за:
  - **Уровень аудио** - должен меняться (зеленая полоса)
  - **Размер буфера** - должен расти

### 6. Сохраните запись
- Нажмите **"💾 Сохранить WAV"**
- Запомните путь к файлу (обычно в Documents/VoiceAssistant/debug)

### 7. Проверьте запись
```powershell
cd apps/desktop
.\check-wav.ps1
```

## 🔍 Что проверять

### ✅ Хорошие признаки
- Уровень аудио: 10-100%
- Размер буфера растет: 50KB → 100KB → 200KB...
- В логах нет ошибок
- Анализ показывает: `[OK] Signal level is good`

### ❌ Проблемы

#### Проблема: Все сэмплы равны нулю
```
[ERROR] ALL SAMPLES ARE ZERO (SILENCE)
```

**Решение:**
1. Выберите устройство с **(Output/Loopback)**
2. Направьте звук на VB-Cable:
   - Откройте "Параметры звука" Windows
   - Установите VB-Cable как устройство по умолчанию
   - Или настройте конкретное приложение на вывод через VB-Cable
3. Проверьте, что VB-Cable включен и не отключен

#### Проблема: Низкий уровень сигнала
```
[WARNING] Very low signal level (< 1%)
```

**Решение:**
1. Увеличьте громкость источника звука
2. Откройте микшер громкости Windows (Win+G)
3. Увеличьте уровень VB-Cable
4. Отключите подавление шума

#### Проблема: Устройство не найдено
```
Device 'VB-Cable' not found
```

**Решение:**
1. Установите VB-Cable: https://vb-audio.com/Cable/
2. Перезагрузите компьютер
3. Перезапустите аудио службу:
   ```powershell
   Restart-Service -Name "Audiosrv" -Force
   ```

## 📊 Компонент отладки

### Секции компонента

1. **Устройства**
   - Список всех аудио устройств
   - Кнопка обновления списка
   - Выбор устройства

2. **Управление захватом**
   - Начать/остановить захват
   - Проверить буфер
   - Сохранить WAV

3. **Статус**
   - Текущий статус (запись/остановлено)
   - Размер буфера в реальном времени
   - Уровень аудио (визуальная полоса)

4. **Логи**
   - История всех операций
   - Ошибки и предупреждения
   - Последние 20 записей

5. **Инструкции**
   - Пошаговое руководство
   - Индикатор режима (анализ вкл/выкл)

### Кнопка анализа

В компоненте отладки есть большая кнопка вверху справа:

- **🤖 Анализ ВКЛЮЧЕН** (зеленая) - аудио отправляется в ChatGPT
- **⏸️ Анализ ОТКЛЮЧЕН** (желтая) - только запись, без ChatGPT

## 🛠️ Скрипты для отладки

### check-wav.ps1
Быстрая проверка последнего сохраненного файла:
```powershell
.\check-wav.ps1
```

Проверка конкретного файла:
```powershell
.\check-wav.ps1 "C:\Users\...\audio_debug_20241104_153045.wav"
```

### diagnose-audio.ps1
Полная диагностика системы:
```powershell
.\diagnose-audio.ps1
```

Проверяет:
- Наличие WAV файлов
- Параметры аудио
- Уровень сигнала
- Аудио устройства
- Аудио службу Windows

## 📁 Где находятся файлы

### Сохраненные записи
```
%USERPROFILE%\Documents\VoiceAssistant\debug\
```

Формат имени:
```
audio_debug_YYYYMMDD_HHMMSS.wav
```

### Логи приложения
```
%USERPROFILE%\AppData\Local\VoiceAssistant\logs\
```

## 🔄 Типичный процесс отладки

1. **Открыть компонент отладки** (кнопка "🔧 Отладка")
2. **Отключить анализ** (кнопка "⏸️ Анализ ВЫКЛ")
3. **Выбрать устройство** (VB-Cable Output/Loopback)
4. **Начать захват** (кнопка "▶️ Начать захват")
5. **Воспроизвести звук** (музыка, видео, разговор)
6. **Проверить уровень** (должен быть > 10%)
7. **Сохранить WAV** (кнопка "💾 Сохранить WAV")
8. **Проанализировать** (`.\check-wav.ps1`)
9. **Исправить проблемы** (если есть)
10. **Включить анализ** (кнопка "🤖 Анализ ВКЛ")
11. **Вернуться к приложению** (кнопка "← Назад")

## 💡 Советы

### Для лучшего качества записи
- Используйте частоту дискретизации 48000 Hz
- Убедитесь, что громкость не на максимуме (избегайте клиппинга)
- Закройте ненужные приложения для снижения нагрузки на CPU

### Для экономии API запросов
- Отключайте анализ когда тестируете
- Включайте только когда нужны ответы от ChatGPT

### Для отладки проблем
- Сохраняйте несколько записей с разными настройками
- Сравнивайте результаты анализа
- Проверяйте логи в реальном времени

## 🆘 Если ничего не помогает

1. Соберите информацию:
   ```powershell
   .\diagnose-audio.ps1 > diagnosis.txt
   ```

2. Сохраните проблемный WAV файл

3. Проверьте логи:
   ```powershell
   Get-Content "$env:USERPROFILE\AppData\Local\VoiceAssistant\logs\*.log" -Tail 50
   ```

4. Проверьте документацию: [docs/audio-debugging.md](../../docs/audio-debugging.md)
