# 🔧 Компонент отладки аудио - Готов к использованию

## ✅ Что добавлено

### 1. Кнопка "🔧 Отладка" в основном приложении
- Расположение: внизу экрана, рядом с "⚙️ Настройки"
- Открывает полноэкранный компонент отладки
- Кнопка "← Назад" для возврата к приложению

### 2. Компонент AudioDebug
Полнофункциональный интерфейс для отладки записи:
- ✅ Список всех аудио устройств
- ✅ Управление захватом (старт/стоп)
- ✅ Мониторинг в реальном времени (уровень + размер буфера)
- ✅ Сохранение WAV файлов
- ✅ Логи всех операций
- ✅ Переключатель анализа (ChatGPT вкл/выкл)
- ✅ Инструкции по использованию

### 3. Скрипты для анализа
- **check-wav.ps1** - быстрая проверка WAV файлов
- **diagnose-audio.ps1** - полная диагностика системы

## 🚀 Как использовать

### Вариант 1: Через кнопку в приложении (РЕКОМЕНДУЕТСЯ)

```powershell
# 1. Запустить приложение
cd apps/desktop
bun run tauri:dev

# 2. Нажать кнопку "🔧 Отладка" внизу экрана

# 3. Следовать инструкциям в компоненте
```

### Вариант 2: Через тестовое приложение

```powershell
# Запустить в режиме тестирования
.\test-audio.ps1
```

## ❌ Типичные проблемы и решения

### Проблема 1: Все сэмплы равны нулю

**Симптомы:**
```
[ERROR] ALL SAMPLES ARE ZERO (SILENCE)
```

**Решение:**
1. В компоненте отладки выберите устройство с **(Output/Loopback)**
2. Откройте "Параметры звука" Windows
3. Установите VB-Cable как устройство по умолчанию
4. Или настройте приложение на вывод через VB-Cable

### Проблема 2: Устройство не найдено

**Симптомы:**
- VB-Cable не в списке устройств
- Ошибка "Device not found"

**Решение:**
1. Установите VB-Cable: https://vb-audio.com/Cable/
2. Перезагрузите компьютер
3. Нажмите "🔄 Обновить список" в компоненте

### Проблема 3: Низкий уровень сигнала

**Симптомы:**
```
[WARNING] Very low signal level (< 1%)
```

**Решение:**
1. Увеличьте громкость источника
2. Откройте микшер Windows (Win+G)
3. Увеличьте уровень VB-Cable
4. Проверьте, что не включено подавление шума
