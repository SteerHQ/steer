## 🎯 Сценарии использования

### 1. Запись с микрофона

**Документация:** [МИКРОФОН-VB-CABLE.md](МИКРОФОН-VB-CABLE.md)

**Устройство:** `CABLE Output (VB-Audio Virtual Cable) (Input)`

**Настройка:**

1. Микрофон → Приложение
2. Приложение → CABLE Input
3. Наше приложение → CABLE Output (Input)

---

### 2. Запись системного звука

**Документация:** [DEVICE-SELECTION.md](DEVICE-SELECTION.md)

**Устройство:** `CABLE Output (VB-Audio Virtual Cable) (Output/Loopback)`

**Настройка:**

1. Установить CABLE Input как устройство по умолчанию
2. Наше приложение → CABLE Output (Output/Loopback)

---

### 3. Запись разговора (оба участника)

**Документация:** [VB-CABLE-MICROPHONE.md](VB-CABLE-MICROPHONE.md) (Сценарий 2)

**Устройство:** `CABLE Output (VB-Audio Virtual Cable) (Output/Loopback)`

**Настройка:**

1. В Zoom/Discord:
   - Микрофон: Ваш физический
   - Динамики: CABLE Input
2. Наше приложение → CABLE Output (Output/Loopback)

---

## ❓ Часто задаваемые вопросы

### В чем разница между (Input) и (Output/Loopback)?

- **(Input)** - записывает звук, который **идет В** VB Cable (микрофон)
- **(Output/Loopback)** - записывает звук, который **идет ИЗ** VB Cable (системный звук)

См. [СХЕМА-VB-CABLE.md](СХЕМА-VB-CABLE.md) для визуального объяснения.

### Почему в файле тишина?

Возможные причины:

1. Выбрано неправильное устройство
2. Звук не идет через VB Cable
3. VB Cable не установлен или не работает

См. [HOW-TO-DEBUG.md](HOW-TO-DEBUG.md) для диагностики.

### Как проверить, что VB Cable работает?

```powershell
.\apps\desktop\check-vb-cable.ps1
```

### Какие альтернативы VB Cable?

См. [AUDIO-ALTERNATIVES.md](AUDIO-ALTERNATIVES.md):

- Stereo Mix (встроен в Windows)
- VoiceMeeter (рекомендуется)
- BlackHole (macOS)
