# 🚨 Быстрое решение: Устройство не найдено

## Ваша ошибка:
```
Ошибка запуска захвата: Audio device 'CABLE Input (VB-Audio Virtual Cable)' not found.
Available devices: Микрофон (Realtek High Definition Audio), CABLE Output (VB-Audio Virtual Cable)
```

## ✅ Решение:

### Выберите правильное устройство из списка!

Доступные устройства:
1. ❌ **Микрофон (Realtek High Definition Audio)** - это микрофон, не подходит
2. ✅ **CABLE Output (VB-Audio Virtual Cable)** - ВОТ ЭТО НУЖНО!

## 📋 Пошаговая инструкция:

### 1. Откройте компонент отладки
Нажмите кнопку **"🔧 Отладка"** в приложении

### 2. Обновите список устройств
Нажмите **"🔄 Обновить список"**

### 3. Найдите в списке:
```
CABLE Output (VB-Audio Virtual Cable) (Output/Loopback)
```
или
```
CABLE Output (VB-Audio Virtual Cable) (Output)
```

### 4. Выберите это устройство
Оно должно быть помечено ✅ зеленым цветом

### 5. Запустите захват
Нажмите **"▶️ Начать захват"**

### 6. Настройте вывод звука на VB-Cable

#### Вариант A: Системный звук (рекомендуется)
1. Откройте **"Параметры звука"** (Win + I → Система → Звук)
2. В разделе **"Вывод"** выберите:
   ```
   CABLE Input (VB-Audio Virtual Cable)
   ```
   ⚠️ Да, для ВЫВОДА звука выбираем "CABLE Input"!
   ⚠️ А для ЗАХВАТА звука выбираем "CABLE Output"!

#### Вариант B: Конкретное приложение
1. Откройте **"Параметры звука"**
2. Прокрутите вниз → **"Дополнительные параметры звука"**
3. Нажмите **"Параметры устройств и громкости приложений"**
4. Найдите приложение (Zoom, Teams, Chrome)
5. Выберите **"CABLE Input (VB-Audio Virtual Cable)"**

### 7. Воспроизведите звук
- Включите музыку
- Откройте видео
- Начните разговор

### 8. Проверьте уровень
В компоненте отладки:
- **Уровень аудио** должен меняться (> 0%)
- **Размер буфера** должен расти

### 9. Сохраните запись
Нажмите **"💾 Сохранить WAV"**

### 10. Проверьте файл
```powershell
cd apps/desktop
.\check-wav.ps1
```

## 🎯 Почему так?

### VB-Cable имеет 2 устройства:

1. **CABLE Input** (устройство воспроизведения)
   - Сюда НАПРАВЛЯЕТСЯ звук
   - Выбирается в настройках вывода
   - Приложения воспроизводят звук СЮДА

2. **CABLE Output** (устройство записи)
   - Отсюда ЗАХВАТЫВАЕТСЯ звук
   - Выбирается в приложении для записи
   - Наше приложение записывает ОТСЮДА

### Схема работы:
```
Приложение (Zoom/Chrome)
    ↓ воспроизводит звук
CABLE Input (устройство воспроизведения)
    ↓ передает звук
CABLE Output (устройство записи)
    ↓ захватывает звук
Наше приложение
```

## ✅ Правильная конфигурация:

### В Windows (для вывода звука):
```
Параметры звука → Вывод → CABLE Input (VB-Audio Virtual Cable)
```

### В нашем приложении (для захвата):
```
Компонент отладки → Устройства → CABLE Output (VB-Audio Virtual Cable)
```

## 🔍 Проверка

После настройки:

1. **В микшере Windows** (Win + G):
   - Найдите "CABLE Input"
   - При воспроизведении звука полоса должна двигаться

2. **В компоненте отладки**:
   - Уровень аудио: 10-100%
   - Размер буфера растет
   - Логи без ошибок

3. **В сохраненном файле**:
   ```powershell
   .\check-wav.ps1
   ```
   Должно быть:
   ```
   [OK] Signal level is good
   ```

## 🆘 Если все еще не работает

### Проблема: CABLE Output не в списке

**Решение:**
1. Переустановите VB-Cable
2. Перезагрузите компьютер
3. Проверьте в "Параметры звука" → "Управление звуковыми устройствами"
4. Убедитесь, что "CABLE Output" включен

### Проблема: Уровень аудио = 0%

**Решение:**
1. Убедитесь, что звук воспроизводится через CABLE Input
2. Проверьте громкость в микшере Windows
3. Попробуйте воспроизвести громкую музыку для теста

### Проблема: Размер буфера не растет

**Решение:**
1. Остановите и запустите захват заново
2. Проверьте логи на наличие ошибок
3. Перезапустите приложение
