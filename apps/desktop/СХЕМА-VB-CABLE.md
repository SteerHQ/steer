# 📊 Схема работы VB Cable

## Визуальная схема

### Вариант 1: Запись с микрофона (Input)

```
┌─────────────┐
│  Микрофон   │
│  (физический)│
└──────┬──────┘
       │
       ▼
┌─────────────────────┐
│  Приложение         │
│  (Zoom, Discord)    │
└──────┬──────────────┘
       │
       ▼
┌─────────────────────┐
│  CABLE Input        │  ← Устройство воспроизведения
│  (VB-Audio)         │
└──────┬──────────────┘
       │
       ▼
┌─────────────────────┐
│  CABLE Output       │
│  (Input) ✅         │  ← Выбрать в приложении
└──────┬──────────────┘
       │
       ▼
┌─────────────────────┐
│  Наше приложение    │
│  (запись)           │
└─────────────────────┘
```

**Что записывается:** Ваш голос с микрофона

### Вариант 2: Запись системного звука (Output/Loopback)

```
┌─────────────────────┐
│  Системный звук     │
│  (музыка, видео)    │
└──────┬──────────────┘
       │
       ▼
┌─────────────────────┐
│  CABLE Input        │  ← Устройство воспроизведения по умолчанию
│  (VB-Audio)         │
└──────┬──────────────┘
       │
       ▼
┌─────────────────────┐
│  CABLE Output       │
│  (Output/Loopback)✅│  ← Выбрать в приложении
└──────┬──────────────┘
       │
       ▼
┌─────────────────────┐
│  Наше приложение    │
│  (запись)           │
└─────────────────────┘
```

**Что записывается:** Все звуки, которые воспроизводятся через VB Cable

### Вариант 3: Запись разговора (оба участника)

```
┌─────────────┐         ┌─────────────┐
│  Микрофон   │         │ Собеседник  │
│  (ваш голос)│         │ (удаленно)  │

       │                       │
       └───────┬───────────────┘
               ▼
       ┌─────────────────────┐
       │  Zoom / Discord     │
       │  Микрофон: Ваш      │
       │  Динамики: CABLE    │
       └──────┬──────────────┘
              │
              ▼
       ┌─────────────────────┐
       │  CABLE Input        │
       │  (VB-Audio)         │
       └──────┬──────────────┘
              │
              ▼
       ┌─────────────────────┐
       │  CABLE Output       │
       │  (Output/Loopback)✅│  ← Выбрать в приложении
       └──────┬──────────────┘
              │
              ▼
       ┌─────────────────────┐
       │  Наше приложение    │
       │  (запись)           │
       └─────────────────────┘
```

**Что записывается:** Весь разговор (вы + собеседник)

## Быстрая справка

| Что нужно записать | Устройство в приложении          | Настройка Windows                     |
| ------------------ | -------------------------------- | ------------------------------------- |
| 🎤 Свой голос      | `CABLE Output (Input)`           | Микрофон → Приложение → CABLE Input   |
| 🔊 Системный звук  | `CABLE Output (Output/Loopback)` | CABLE Input = устройство по умолчанию |
| 💬 Разговор (оба)  | `CABLE Output (Output/Loopback)` | Zoom: Динамики = CABLE Input          |
| 🎵 Музыка/видео    | `CABLE Output (Output/Loopback)` | CABLE Input = устройство по умолчанию |

## Проверка настройки

### 1. Откройте микшер громкости Windows

Нажмите `Win + G` или щелкните правой кнопкой по значку звука → "Открыть микшер громкости"

### 2. Проверьте, что звук идет через VB Cable

Найдите **CABLE Input** в списке устройств. При воспроизведении звука полоса должна двигаться.

### 3. В приложении проверьте уровень аудио

Откройте компонент отладки:

- Уровень аудио должен быть > 0%
- Размер буфера должен расти
- Полоса уровня должна двигаться

## Типичные ошибки

### ❌ Выбрано Input, но нужен системный звук

**Проблема:** Выбрали `CABLE Output (Input)`, но хотите записать музыку/видео

**Решение:** Выберите `CABLE Output (Output/Loopback)`

### ❌ Выбрано Output/Loopback, но нужен микрофон

**Проблема:** Выбрали `CABLE Output (Output/Loopback)`, но хотите записать только свой голос

**Решение:** Выберите `CABLE Output (Input)` и настройте микрофон на вывод в CABLE Input

### ❌ Звук не идет через VB Cable

**Проблема:** Уровень аудио = 0%, буфер не растет

**Решение:**

1. Откройте "Параметры звука"
2. Установите CABLE Input как устройство воспроизведения
3. Или настройте конкретное приложение на вывод через CABLE Input
