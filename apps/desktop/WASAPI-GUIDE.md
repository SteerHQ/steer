# WASAPI Loopback - Захват системного аудио без виртуальных устройств

## Что это?

**WASAPI Loopback** - это встроенная функция Windows, которая позволяет захватывать системное аудио **без установки виртуальных аудиоустройств** (VB-Cable, VoiceMeeter, Stereo Mix).

## Преимущества

✅ **Не требует установки драйверов** - работает "из коробки" на Windows Vista и выше  
✅ **Захватывает все системное аудио** - Zoom, Google Meet, YouTube, любые приложения  
✅ **Более стабильно** - нет проблем с драйверами виртуальных устройств  
✅ **Лучшее качество** - прямой доступ к аудио потоку Windows  
✅ **Проще в использовании** - не нужно настраивать маршрутизацию аудио  

## Как использовать

### 1. Запустите приложение

Откройте Voice Assistant Overlay

### 2. Выберите устройство

В списке аудиоустройств выберите:
```
WASAPI Loopback (System Audio - No Virtual Device Needed)
```

### 3. Начните захват

Нажмите "Начать захват" - приложение автоматически начнет записывать все системное аудио.

### 4. Используйте Zoom/Google Meet

Просто начните звонок в Zoom или Google Meet - голос собеседника будет автоматически захватываться.

## Как это работает

```
Zoom/Google Meet → Windows Audio Engine → WASAPI Loopback → Voice Assistant
```

WASAPI Loopback подключается напрямую к аудио движку Windows и получает копию всего аудио, которое воспроизводится в системе.

## Технические детали

### Что захватывается?

WASAPI Loopback захватывает:
- ✅ Голос собеседников в Zoom/Google Meet
- ✅ Системные звуки
- ✅ Музыку и видео
- ✅ Любое аудио, воспроизводимое через динамики/наушники

### Что НЕ захватывается?

- ❌ Ваш собственный микрофон (только то, что слышите вы)
- ❌ Аудио из других физических устройств

### Формат аудио

- **Частота дискретизации**: Автоматически определяется из системных настроек (обычно 48000 Hz)
- **Формат**: 16-bit PCM Mono
- **Буфер**: До 10 секунд аудио

## Требования

- Windows Vista или выше (рекомендуется Windows 10/11)
- Любое аудиоустройство (встроенное или внешнее)

## Устранение проблем

### Проблема: Не захватывается аудио

**Решение:**
1. Убедитесь, что звук воспроизводится в системе (проверьте громкость)
2. Проверьте, что выбрано устройство "WASAPI Loopback"
3. Перезапустите захват

### Проблема: Низкое качество звука

**Решение:**
1. Увеличьте системную громкость
2. Проверьте настройки качества звука в Windows:
   - Параметры → Система → Звук → Свойства устройства → Дополнительно
   - Установите максимальное качество (48000 Hz, 24 bit)

### Проблема: Задержка в захвате

**Решение:**
- WASAPI Loopback имеет минимальную задержку (~10-20ms)
- Это нормально и не влияет на качество распознавания речи

## Для разработчиков

### Архитектура

```rust
// Создание WASAPI capture
let capture = AudioCapture::new("WASAPI Loopback")?;

// Начать захват
capture.start_capture()?;

// Получить аудио данные
let buffer = capture.get_buffer();

// Остановить захват
capture.stop_capture()?;
```

### Реализация

- Использует Windows API напрямую через `windows-rs`
- Работает в отдельном потоке для минимальной задержки
- Автоматически конвертирует float32 в int16 PCM
- Поддерживает любое количество каналов (конвертирует в mono)

### Код

Реализация находится в:
- `src-tauri/src/audio_wasapi.rs` - WASAPI capture
- `src-tauri/src/audio.rs` - Общий интерфейс

## FAQ

**Q: Нужно ли устанавливать VB-Cable?**  
A: Нет! WASAPI Loopback работает без виртуальных устройств.

**Q: Работает ли на Windows 7?**  
A: Да, WASAPI Loopback доступен с Windows Vista.

**Q: Можно ли захватывать только Zoom, без других звуков?**  
A: Нет, WASAPI Loopback захватывает все системное аудио. Для выборочного захвата используйте VoiceMeeter.

**Q: Влияет ли это на производительность?**  
A: Минимально. WASAPI очень эффективен и использует <1% CPU.

**Q: Можно ли использовать с другими приложениями?**  
A: Да! Работает с любыми приложениями: Discord, Skype, Teams, игры, браузеры.
