# 🎯 Выбор правильного устройства

## ❌ ПРОБЛЕМА: Нет звука в файле

Если вы видите:
```
[2:28:45 PM] Запуск захвата с устройства: CABLE Output (VB-Audio Virtual Cable) (Input)
[2:28:46 PM] Захват запущен успешно
```

Но в файле нет звука - **вы выбрали неправильное устройство!**

## 🔍 Типы устройств

### ❌ НЕПРАВИЛЬНО: (Input)
```
CABLE Output (VB-Audio Virtual Cable) (Input)
                                       ^^^^^^
                                       НЕ ТО!
```

**Что это:**
- Это микрофонный вход
- Захватывает звук С микрофона
- НЕ захватывает системный звук
- НЕ захватывает разговоры из приложений

**Результат:**
- Файл создается
- Но в нем ТИШИНА (все сэмплы = 0)
- Уровень аудио = 0%

### ✅ ПРАВИЛЬНО: (Output/Loopback)
```
CABLE Output (VB-Audio Virtual Cable) (Output/Loopback)
                                       ^^^^^^^^^^^^^^^^
                                       ВОТ ЭТО!
```

**Что это:**
- Это loopback устройство
- Захватывает звук, который ВОСПРОИЗВОДИТСЯ через устройство
- Захватывает системный звук
- Захватывает разговоры из приложений

**Результат:**
- Файл создается
- В нем есть ЗВУК
- Уровень аудио меняется

## 📋 Как выбрать правильное устройство

### В компоненте отладки

1. Откройте список устройств
2. Найдите устройство с **(Output/Loopback)**
3. Оно будет помечено ✅ и выделено зеленым
4. Устройства с **(Input)** помечены ❌ и красным

```
Список устройств:
┌────────────────────────────────────────────────────────┐
│ ❌ CABLE Output (VB-Audio Virtual Cable) (Input)      │  ← НЕ ТО!
│ ✅ CABLE Output (VB-Audio Virtual Cable) (Output/Loop │  ← ВОТ ЭТО!
│ ❌ Microphone (Realtek Audio) (Input)                 │
│    Speakers (Realtek Audio) (Output)                  │
└────────────────────────────────────────────────────────┘
```

### После выбора

Компонент покажет подсказку:

**Если выбрали Input:**
```
┌────────────────────────────────────────────────────────┐
│ ❌ НЕПРАВИЛЬНОЕ УСТРОЙСТВО!                            │
│ Вы выбрали Input устройство - оно НЕ захватывает      │
│ системный звук.                                        │
│ Выберите устройство с (Output/Loopback)               │
└────────────────────────────────────────────────────────┘
```

**Если выбрали Output/Loopback:**
```
┌────────────────────────────────────────────────────────┐
│ ✅ ПРАВИЛЬНОЕ УСТРОЙСТВО!                              │
│ Это устройство поддерживает захват системного звука.  │
└────────────────────────────────────────────────────────┘
```

## 🎧 Настройка VB-Cable

### Шаг 1: Установка VB-Cable как устройства по умолчанию

1. Откройте **"Параметры звука"** (Win + I → Система → Звук)
2. В разделе **"Вывод"** выберите **"CABLE Input (VB-Audio Virtual Cable)"**
3. Это направит весь системный звук через VB-Cable

### Шаг 2: Настройка конкретного приложения

Если не хотите менять устройство по умолчанию:

1. Откройте **"Параметры звука"**
2. Прокрутите вниз до **"Дополнительные параметры звука"**
3. Нажмите **"Параметры устройств и громкости приложений"**
4. Найдите нужное приложение (Zoom, Teams, Chrome и т.д.)
5. В выпадающем списке выберите **"CABLE Input (VB-Audio Virtual Cable)"**

### Шаг 3: Проверка

1. Воспроизведите звук (музыку, видео)
2. В компоненте отладки уровень аудио должен меняться
3. Размер буфера должен расти

## 🔧 Отладка

### Проверка 1: Правильное устройство выбрано?

В логах должно быть:
```
[14:32:15] Запуск захвата с устройства: CABLE Output (VB-Audio Virtual Cable)
                                         ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                                         БЕЗ (Input) в конце!
```

### Проверка 2: Звук идет через VB-Cable?

1. Откройте микшер громкости Windows (Win + G)
2. Найдите **"CABLE Input"**
3. При воспроизведении звука полоса должна двигаться

### Проверка 3: Уровень аудио меняется?

В компоненте отладки:
```
Уровень аудио: 45.3%
████████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░
```

Полоса должна двигаться при воспроизведении звука.

### Проверка 4: Буфер растет?

```
Размер буфера: 245760 байт (240 KB)
```

Число должно увеличиваться каждые несколько секунд.

## 🎯 Быстрое решение

### Если у вас нет звука:

1. **Откройте компонент отладки** (кнопка "🔧 Отладка")
2. **Остановите захват** (если запущен)
3. **Найдите устройство с (Output/Loopback)**
4. **Выберите его** (должна появиться зеленая галочка ✅)
5. **Запустите захват заново**
6. **Воспроизведите звук**
7. **Проверьте уровень** (должен быть > 0%)

### Если устройства с (Output/Loopback) нет:

1. Переустановите VB-Cable
2. Перезагрузите компьютер
3. Проверьте, что драйвер установлен:
   ```powershell
   Get-PnpDevice -Class "AudioEndpoint" | Where-Object {$_.FriendlyName -like "*Cable*"}
   ```
