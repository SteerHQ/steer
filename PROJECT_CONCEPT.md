# Voice Assistant Overlay - Концепция проекта

## Главная идея

AI-ассистент для прохождения технических собеседований на позицию Software Engineer.

## Как это работает

### 1. Захват вопросов
```
Интервьюер в Zoom/Teams/Meet
         ↓
   Системное аудио (WASAPI Loopback)
         ↓
   Захват аудио потока
         ↓
   Транскрипция через Whisper API
         ↓
   "Расскажите о вашем опыте с React"
```

### 2. Генерация ответов
```
Вопрос интервьюера
         ↓
   GPT-4o API
         ↓
   Готовый ответ (3-5 предложений)
         ↓
   "Я работаю с React уже более 3 лет..."
```

### 3. Зачитывание ответа
```
Кандидат видит готовый ответ
         ↓
   Зажимает SPACE (приостановка обработки)
         ↓
   Зачитывает ответ интервьюеру
         ↓
   Отпускает SPACE
         ↓
   Система снова слушает вопросы
```

## Ключевые особенности

### ✅ Что делает система

1. **Захватывает вопросы** интервьюера из системного аудио
2. **Транскрибирует** вопросы через OpenAI Whisper
3. **Генерирует готовые ответы** через GPT-4o
4. **Показывает ответы** в истории чата
5. **Приостанавливает обработку** когда кандидат отвечает

### ❌ Что НЕ делает система

1. **НЕ записывает** речь кандидата
2. **НЕ отправляет** ответы кандидата в API
3. **НЕ анализирует** качество ответов кандидата
4. **НЕ озвучивает** ответы (кандидат зачитывает сам)

## Технический стек

### Desktop App (Tauri)
- **Frontend**: React + TypeScript + Vite
- **Backend**: Rust + cpal (audio capture)
- **Audio**: WASAPI Loopback для Windows

### API Server
- **Framework**: Hono.js (Node.js)
- **AI**: OpenAI Whisper + GPT-4o
- **Format**: Binary WAV для эффективности

### Ключевые библиотеки
- `@tauri-apps/api` - Tauri bindings
- `cpal` - Cross-platform audio library (Rust)
- `hono` - Web framework
- `@ai-sdk/openai` - OpenAI SDK
- `zustand` - State management

## Workflow пользователя

### Подготовка
1. Запустить API сервер (`npm run dev` в apps/api)
2. Настроить OPENAI_API_KEY в apps/api/.env
3. Запустить desktop app (`npm run tauri dev`)
4. Выбрать режим "Interview"

### Во время интервью
1. Интервьюер задает вопрос в Zoom
2. Система показывает вопрос в чате
3. Через 2-3 секунды появляется готовый ответ
4. Кандидат зажимает SPACE
5. Зачитывает ответ (можно адаптировать)
6. Отпускает SPACE
7. Система слушает следующий вопрос

### Пример сессии
```
[10:15:23] ❓ Вопрос: "Что такое React hooks?"
[10:15:26] ✅ Ответ: "React hooks - это функции, которые 
           позволяют использовать состояние и другие возможности 
           React без классов. Я активно использую их в своих 
           проектах..."

[10:16:45] ❓ Вопрос: "Расскажите о вашем опыте с TypeScript"
[10:16:48] ✅ Ответ: "Я работаю с TypeScript уже более 3 лет 
           и считаю его незаменимым для больших проектов..."
```

## Преимущества

### Для кандидата
✅ Готовые технически правильные ответы
✅ Уверенность на собеседовании
✅ Структурированные ответы с примерами
✅ Экономия времени на подготовку

### Технические

✅ Работает локально (приватность)
✅ Эффективная передача данных (binary WAV)
✅ Простой UI (не отвлекает)

## Ограничения

⚠️ **Этические соображения**
- Использование AI-ассистента на собеседовании может быть неэтичным
- Некоторые компании могут считать это обманом
- Рекомендуется использовать только для тренировки

⚠️ **Технические ограничения**
- Требует Windows (WASAPI Loopback)
- Требует стабильного интернета (OpenAI API)
- Задержка 2-3 секунды на обработку
- Стоимость API запросов
