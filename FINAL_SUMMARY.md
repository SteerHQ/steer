## 1. Pre-roll буфер (захват начала речи)

### Проблема
Первое слово интервьюера не попадало в запись.

### Решение
Добавлен кольцевой буфер на **3 секунды**, который постоянно хранит последние аудио данные.

### Технические детали
- Фоновый WASAPI поток работает постоянно
- Pre-roll буфер: 3 секунды (288,000 байт)
- При `start_capture()` pre-roll копируется в main buffer

### Файлы
- `apps/desktop/src-tauri/src/audio_wasapi.rs`

### Документация
- `AUDIO_CAPTURE_IMPROVEMENTS.md`

---

## 2. Накопление речи и детекция пауз

### Проблема
Один вопрос разбивался на несколько частей из-за обработки каждые 5 секунд.

### Решение
Система накапливает аудио во время речи и обрабатывает только после **2 секунд тишины**.

### Технические детали
- Мониторинг каждые 5 секунд
- Порог голоса: 0.02 (2%, настраивается)
- Длительность паузы: 2 секунды
- Состояния: idle → speaking → paused → processing

### Файлы
- `apps/desktop/src/App.tsx`

### Документация
- `SPEECH_ACCUMULATION.md`

## 3. Post-roll задержка (захват конца речи)

### Проблема
Последнее слово вопроса не попадало в запись.

### Решение
Добавлена задержка **500ms** после обнаружения паузы перед получением данных из буфера.

### Технические детали
```typescript
await new Promise(resolve => setTimeout(resolve, 500));
```

### Файлы
- `apps/desktop/src/App.tsx`

### Документация
- `POST_ROLL_FIX.md`

## 5. Event-Driven архитектура

### Проблема
AudioVisualizer создавал 10 IPC вызовов в секунду (polling).

### Решение
Переход на **Tauri Events** (push модель).

### Технические детали
- Rust запускает фоновый поток
- Поток отправляет события `audio-level` каждые 100ms
- Фронтенд слушает события через `listen()`

### Преимущества
- Нет IPC запросов (экономия 99%)
- CPU usage -75%
- Latency -80%

### Файлы
- `apps/desktop/src-tauri/src/commands.rs` - emitter команды
- `apps/desktop/src-tauri/src/main.rs` - регистрация
- `apps/desktop/src/components/audio-visualizer.tsx` - использование событий

### Документация
- `EVENT_DRIVEN_ARCHITECTURE.md`
- `COMPILATION_FIX.md`

## Архитектура системы

```
┌─────────────────────────────────────────────────────────────┐
│  WASAPI Background Thread (всегда работает)                 │
│  ├─► Pre-roll Buffer (3 сек) - кольцевой буфер             │
│  └─► Main Buffer (до 30 сек) - при активной записи         │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  Audio Level Emitter Thread (фоновый)                      │
│  ├─► Вычисляет RMS каждые 100ms                            │
│  └─► Отправляет события "audio-level"                      │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  Мониторинг речи (каждые 5 секунд)                         │
│  ├─► get_audio_level() - проверка без очистки               │
│  ├─► Голос? → Накапливаем (состояние: "Говорите")          │
│  ├─► Тишина? → Ждем 2 сек (состояние: "Пауза")             │
│  ├─► 2 сек тишины → Ждем 500ms (post-roll)                 │
│  └─► Post-roll → Обработка (состояние: "Слушаю")           │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  Обработка накопленного аудио                               │
│  ├─► Whisper API - транскрипция                             │
│  ├─► GPT-5-mini - детекция вопроса                          │
│  └─► GPT-5 - генерация ответа                               │
└─────────────────────────────────────────────────────────────┘
```

---

## Параметры системы

| Компонент | Параметр | Значение | Настраивается |
|-----------|----------|----------|---------------|
| Pre-roll | Размер буфера | 3 секунды | Нет |
| Post-roll | Задержка | 500ms | Нет |
| Main buffer | Максимум | 30 секунд | Нет |
| VAD | Порог голоса | 0.02 (2%) | Да (UI) |
| Паузы | Длительность | 2 секунды | Нет |
| Мониторинг | Интервал | 5 секунд | Нет |
| Events | Интервал | 100ms | Нет |
| RMS анализ | Окно | 0.5 секунды | Нет |
