# Накопление речи и детекция пауз

## Проблема
При тестировании выяснилось, что один вопрос разбивался на несколько файлов из-за того, что обработка происходила каждые 5 секунд независимо от того, закончил ли говорить человек.

## Решение

### Архитектура

```
┌─────────────────────────────────────────────────────────┐
│  Мониторинг каждые 5 секунд                             │
└─────────────────┬───────────────────────────────────────┘
                  │
                  ├──► get_audio_level() - проверка уровня
                  │    (без очистки буфера)
                  │
                  ├──► Голос обнаружен?
                  │    ├─ ДА → Продолжаем накапливать
                  │    └─ НЕТ → Проверяем паузу
                  │
                  └──► Пауза 2 секунды?
                       ├─ ДА → Обрабатываем весь буфер
                       └─ НЕТ → Ждем еще
```

### Состояния речи

1. **Тишина** - нет активной речи, ждем начала
2. **Активная речь** - голос обнаружен, накапливаем аудио
3. **Пауза** - речь закончилась, отсчитываем 2 секунды
4. **Обработка** - пауза завершена, обрабатываем накопленное аудио

### Параметры

| Параметр | Значение | Описание |
|----------|----------|----------|
| Интервал проверки | 5 секунд | Как часто проверяем уровень аудио |
| Порог голоса | 0.02 (2%) | Минимальный уровень для детекции речи |
| Длительность паузы | 2 секунды | Сколько тишины ждать перед обработкой |
| Pre-roll буфер | 3 секунды | Захват начала речи |
| Post-roll задержка | 500ms | Захват конца речи перед получением данных |

### Post-roll задержка

После обнаружения 2 секунд тишины система ждет дополнительные **500ms** перед получением данных из буфера. Это гарантирует, что последние слова успеют записаться в буфер.

```typescript
// Wait 500ms to capture the tail of speech (post-roll)
await new Promise(resolve => setTimeout(resolve, 500));
```

### Преимущества

✅ **Целостность вопросов** - вопрос не разбивается на части  
✅ **Полный захват речи** - начало (pre-roll) и конец (post-roll) не теряются  
✅ **Лучшая транскрипция** - больше контекста для Whisper  
✅ **Меньше ложных срабатываний** - игнорируем короткие звуки  
✅ **Естественная обработка** - как человек воспринимает речь

### Пример работы

```
Время | Событие                    | Действие
------|----------------------------|---------------------------
0s    | Тишина                     | Ждем
2s    | Голос обнаружен (3%)       | Начинаем накапливать
4s    | Голос продолжается (5%)    | Продолжаем накапливать
6s    | Голос продолжается (4%)    | Продолжаем накапливать
8s    | Тишина (0.5%)              | Начинаем отсчет паузы
10s   | Тишина (0.3%)              | Пауза 2 секунды
10.5s | Post-roll задержка         | Ждем 500ms для захвата конца
11s   | Получение данных           | ОБРАБОТКА
```

### Код

**Состояние:**
```typescript
const lastVoiceTimeRef = useRef<number>(0);
const silenceStartTimeRef = useRef<number>(0);
const isSpeechActiveRef = useRef<boolean>(false);
```

**Логика:**
```typescript
// Голос обнаружен
if (audioLevel >= VOICE_THRESHOLD) {
  lastVoiceTimeRef.current = now;
  if (!isSpeechActiveRef.current) {
    isSpeechActiveRef.current = true;
    silenceStartTimeRef.current = 0;
  }
  return; // Продолжаем накапливать
}

// Тишина обнаружена
if (isSpeechActiveRef.current) {
  if (silenceStartTimeRef.current === 0) {
    silenceStartTimeRef.current = now;
  }
  
  const silenceDuration = now - silenceStartTimeRef.current;
  if (silenceDuration < 2000) {
    return; // Ждем еще
  }
  
  // Обрабатываем накопленное аудио
  processAccumulatedAudio();
}
```

### Настройка

Пользователь может настроить порог голоса через UI:
- Компонент `VoiceSensitivity` в режиме интервью
- Значение сохраняется в `localStorage` как `voice_threshold`
- По умолчанию: 0.02 (2%)
