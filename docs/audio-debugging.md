# Отладка записи аудио

Руководство по отладке проблем с записью разговора через аудио устройства.

## Быстрый старт

### 1. Использование компонента отладки

Добавьте компонент `AudioDebug` в ваше приложение:

```tsx
import { AudioDebug } from "./components";

function App() {
  return (
    <div>
      <AudioDebug />
    </div>
  );
}
```

### 2. Проверка устройств

1. Откройте компонент отладки
2. Нажмите "Обновить список" для загрузки доступных устройств
3. Проверьте наличие виртуального аудио устройства

**Рекомендуемые устройства (выберите одно):**

#### Вариант 1: VoiceMeeter (РЕКОМЕНДУЕТСЯ)

- Бесплатный и стабильный
- Скачать: https://vb-audio.com/Voicemeeter/
- В списке: `VoiceMeeter Output` или `VoiceMeeter Input`

#### Вариант 2: Stereo Mix (встроенный в Windows)

- Уже установлен в Windows
- Включить: Параметры звука → Управление звуковыми устройствами → Stereo Mix → Включить
- В списке: `Stereo Mix`

#### Вариант 3: VB-Cable (если уже установлен)

- `VB-Cable (Output/Loopback)` - для захвата системного аудио
- `VB-Cable (Input)` - для записи с микрофона

### 3. Тестирование записи

1. Выберите устройство из списка
2. Нажмите "Начать захват"
3. Воспроизведите звук через выбранное устройство
4. Наблюдайте за:
   - Уровнем аудио (должен меняться при воспроизведении)
   - Размером буфера (должен расти)
5. Нажмите "Сохранить WAV" для сохранения записи

### 4. Анализ записи

Используйте PowerShell скрипт для анализа сохраненного файла:

```powershell
# Анализ последнего сохраненного файла
.\apps\desktop\analyze-audio.ps1

# Анализ конкретного файла
.\apps\desktop\analyze-audio.ps1 "C:\Users\...\audio_debug_20241104_153045.wav"
```

## Типичные проблемы

### Проблема 1: Устройство не найдено

**Симптомы:**

- Виртуальное аудио устройство не отображается в списке
- Ошибка "Device not found"

**Решение:**

#### Вариант 1: Использовать Stereo Mix (самый простой)

1. Откройте "Параметры звука" → "Управление звуковыми устройствами"
2. Найдите "Stereo Mix" и нажмите "Включить"
3. Перезапустите приложение

#### Вариант 2: Установить VoiceMeeter (рекомендуется)

1. Скачайте: https://vb-audio.com/Voicemeeter/
2. Установите и перезагрузите компьютер
3. Запустите VoiceMeeter один раз для инициализации
4. Перезапустите аудио службу:
   ```powershell
   Restart-Service -Name "Audiosrv" -Force
   ```

#### Вариант 3: Переустановить VB-Cable (если используете)

1. Удалите текущую версию
2. Скачайте заново: https://vb-audio.com/Cable/
3. Установите и перезагрузите компьютер

### Проблема 2: Буфер пустой

**Симптомы:**

- Размер буфера = 0 байт
- Уровень аудио = 0%
- Сохраненный WAV файл содержит только тишину

**Возможные причины:**

#### A. Неправильное устройство

- Выбрано устройство "Input" вместо "Output/Loopback"
- **Решение:** Выберите устройство для захвата вывода (Output/Loopback/Stereo Mix)

#### B. Звук не направлен на устройство

- Приложение воспроизводит звук через другое устройство
- **Решение для Stereo Mix:**
  1. Stereo Mix захватывает ВСЕ звуки системы автоматически
  2. Просто воспроизведите любой звук
- **Решение для VoiceMeeter/VB-Cable:**
  1. Откройте "Параметры звука" → "Дополнительные параметры звука"
  2. Установите виртуальное устройство как устройство по умолчанию
  3. Или настройте конкретное приложение на вывод через него

#### C. Проблема с драйвером

- Драйвер не работает корректно
- **Решение:**
  1. Попробуйте Stereo Mix (встроенный в Windows)
  2. Или переустановите VoiceMeeter/VB-Cable
  3. Используйте версию драйвера, совместимую с вашей ОС

### Проблема 3: Низкий уровень сигнала

**Симптомы:**

- Уровень аудио < 10%
- Анализ показывает "Низкий уровень сигнала"

**Решение:**

1. Увеличьте громкость источника звука
2. Проверьте уровень громкости устройства в микшере Windows
3. Для Stereo Mix: увеличьте общую громкость системы
4. Для VoiceMeeter: настройте уровни в приложении VoiceMeeter
5. Убедитесь, что не включено подавление шума

### Проблема 4: Прерывистая запись

**Симптомы:**

- Буфер периодически обнуляется
- Запись содержит пропуски

**Возможные причины:**

#### A. Высокая нагрузка на CPU

- **Решение:** Закройте ненужные приложения

#### B. Проблемы с потоком

- **Решение:** Проверьте логи Rust на наличие ошибок потока

#### C. Переполнение буфера

- Буфер ограничен 10 секундами (960KB)
- **Решение:** Увеличьте `MAX_BUFFER_SIZE` в `audio.rs`

## Расширенная отладка

### Логирование Rust

Включите детальное логирование в `src-tauri/src/logger.rs`:

```rust
// Установите уровень TRACE для максимальной детализации
.with_max_level(tracing::Level::TRACE)
```

Логи сохраняются в:

```
%USERPROFILE%\AppData\Local\VoiceAssistant\logs\
```

### Проверка конфигурации устройства

Добавьте в `audio.rs` вывод конфигурации:

```rust
tracing::info!("Device config: {:?}", config);
tracing::info!("Sample rate: {}", config.sample_rate().0);
tracing::info!("Channels: {}", config.channels());
```

### Мониторинг в реальном времени

Используйте компонент `AudioDebug` для мониторинга:

- Уровень аудио обновляется каждые 100ms
- Размер буфера показывает накопление данных
- Логи отображают все операции

## Анализ WAV файлов

Скрипт `analyze-audio.ps1` предоставляет:

### Базовая информация

- Размер файла
- Формат (PCM)
- Количество каналов (Mono/Stereo)
- Частота дискретизации (Hz)

### Анализ сигнала

- Максимальная амплитуда
- RMS уровень
- Процент ненулевых сэмплов

### Диагностика

- ✅ Уровень сигнала в норме
- ⚠️ Низкий уровень сигнала
- ❌ Все сэмплы равны нулю (тишина)

## Чек-лист отладки

Перед обращением за помощью, проверьте:

- [ ] VB-Cable установлен и отображается в списке устройств
- [ ] Выбрано правильное устройство (Output/Loopback)
- [ ] Звук воспроизводится через VB-Cable
- [ ] Уровень громкости VB-Cable не на минимуме
- [ ] Буфер накапливает данные (размер > 0)
- [ ] Уровень аудио меняется при воспроизведении
- [ ] Сохраненный WAV файл содержит звук (не тишину)
- [ ] Проверены логи на наличие ошибок

## Полезные команды

### Проверка аудио служб Windows

```powershell
Get-Service -Name "Audiosrv" | Select-Object Status, StartType
```

### Список аудио устройств через PowerShell

```powershell
Get-PnpDevice -Class "AudioEndpoint" | Where-Object {$_.Status -eq "OK"}
```

### Очистка папки отладки

```powershell
Remove-Item "$env:USERPROFILE\Documents\VoiceAssistant\debug\*.wav"
```
