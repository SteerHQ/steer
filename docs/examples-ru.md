## Типичные сценарии использования

### 1. Теоретические вопросы по JavaScript

**Режим:** 🎯 Собеседование

**Вопросы:**
- "Что такое Event Loop?"
- "Объясни разницу между var, let и const"
- "Что такое прототипное наследование?"
- "Как работает this в JavaScript?"

**Ожидаемые ответы:**
Краткие (2-3 предложения), фокус на ключевых концепциях, простой язык.

### 8. Coding Challenge - Массивы

**Режим:** 🧮 Алгоритмы

**Задачи:**
```
1. "Найти максимальный элемент в массиве"
   → O(n), линейный поиск

2. "Отсортировать массив"
   → O(n log n), quicksort или mergesort

3. "Найти дубликаты в массиве"
   → O(n), используй Set или HashMap

4. "Развернуть массив in-place"
   → O(n), два указателя с концов
```

---

### 9. Coding Challenge - Строки

**Режим:** 🧮 Алгоритмы

**Задачи:**
```
1. "Проверить анаграммы"
   → O(n), сортировка или HashMap

2. "Найти первый неповторяющийся символ"
   → O(n), HashMap для подсчета

3. "Сжать строку (aaabbc → a3b2c1)"
   → O(n), итерация с подсчетом

4. "Проверить валидность скобок"
   → O(n), используй Stack
```

## Советы по эффективному использованию

### ✅ Делайте

1. **Формулируйте вопросы четко и конкретно**
   - ✅ "Что такое closure в JavaScript?"
   - ❌ "Расскажи про JavaScript"

2. **Используйте правильный режим**
   - Теория → Собеседование
   - Задачи → Алгоритмы
   - Факты → Шпаргалка

3. **Задавайте связанные вопросы в режиме собеседования**
   - Ассистент помнит контекст
   - Ответы становятся более релевантными

4. **Очищайте историю при смене темы**
   - Избегайте путаницы в контексте
   - Начинайте новую тему с чистого листа

### ❌ Избегайте

1. **Слишком широких вопросов**
   - ❌ "Расскажи всё про React"
   - ✅ "Что такое React hooks?"

2. **Многосоставных вопросов**
   - ❌ "Что такое Promise и async/await и как они работают вместе?"
   - ✅ "Что такое Promise?" → "Разница между Promise и async/await?"

3. **Неконкретных формулировок**
   - ❌ "Помоги с задачей"
   - ✅ "Как найти максимальный элемент в массиве?"

## Частые вопросы

**Q: Сколько вопросов помнит ассистент?**
A: В режиме собеседования - последние 10 пар вопрос-ответ.

**Q: Можно ли использовать на английском?**
A: Да, но оптимизировано для русского языка.

**Q: Как быстро переключать режимы?**
A: Клик по текущему режиму → выбор нового режима.

**Q: Что делать, если ответ слишком длинный?**
A: Попробуйте режим шпаргалки или переформулируйте вопрос конкретнее.

**Q: Можно ли сохранить историю вопросов?**
A: Пока нет, но планируется в будущих версиях.
