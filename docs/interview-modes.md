# Режимы помощи при собеседовании

Voice Assistant Overlay поддерживает несколько специализированных режимов для помощи на технических собеседованиях.

## Режимы работы

### 🎯 Режим собеседования (Interview Mode)

**Когда использовать:**
- Во время технического собеседования
- Когда нужны краткие, точные ответы
- Когда важен контекст предыдущих вопросов

**Особенности:**
- Краткие ответы (2-3 предложения)
- Сохраняет контекст последних 10 вопросов
- Фокус на ключевых концепциях и определениях
- Простой, понятный язык

**Пример:**
```
Вопрос: "Что такое замыкание в JavaScript?"
Ответ: "Замыкание - это функция, которая имеет доступ к переменным 
из внешней области видимости, даже после завершения выполнения 
внешней функции. Используется для инкапсуляции данных и создания 
приватных переменных."
```

### 🧮 Режим алгоритмов (Algorithm Mode)

**Когда использовать:**
- При решении алгоритмических задач
- Когда нужно понять подход к решению
- Для оценки сложности алгоритма

**Особенности:**
- Объяснение подхода к решению (1-2 предложения)
- Временная и пространственная сложность
- Ключевая структура данных или алгоритм
- Краткий псевдокод (максимум 5 строк)

**Пример:**
```
Вопрос: "Как найти два числа в массиве, сумма которых равна target?"
Ответ: "Используй HashMap для хранения чисел. Для каждого элемента 
проверяй, есть ли (target - element) в HashMap. Сложность: O(n) время, 
O(n) память.

Псевдокод:
map = {}
for num in array:
  if (target - num) in map: return [num, target-num]
  map[num] = true"
```

### 📝 Режим шпаргалки (Cheatsheet Mode)

**Когда использовать:**
- Для быстрого напоминания синтаксиса
- Когда нужны только факты
- Для определений и формул

**Особенности:**
- Максимально краткие ответы (1 предложение или список)
- Только факты, без объяснений
- Синтаксис, определения, формулы

**Пример:**
```
Вопрос: "Методы массива в JavaScript"
Ответ: "map, filter, reduce, forEach, find, some, every, slice, splice, 
concat, join, push, pop, shift, unshift"
```

### 💬 Общий режим (General Mode)

**Когда использовать:**
- Для обычных технических вопросов
- Когда не нужен специализированный режим
- Для общих объяснений

**Особенности:**
- Стандартные технические ответы
- Баланс между краткостью и полнотой
- Без специальной оптимизации

## Управление контекстом

### История вопросов

В режиме собеседования ассистент сохраняет последние 10 пар вопрос-ответ. 
Это позволяет:

- Давать более релевантные ответы
- Учитывать контекст беседы
- Избегать повторений

### Очистка истории

Чтобы очистить историю вопросов:

1. Откройте выпадающее меню режимов
2. Нажмите "🗑️ Очистить историю (N)"

История автоматически очищается при:
- Переключении на другой режим
- Перезапуске приложения

## Советы по использованию

### Для собеседований

1. **Начните с режима собеседования** - он оптимизирован для кратких ответов
2. **Переключайтесь на режим алгоритмов** - когда начинается coding challenge
3. **Используйте шпаргалку** - для быстрого напоминания синтаксиса

### Формулировка вопросов

**Хорошо:**
- "Что такое Promise в JavaScript?"
- "Как работает виртуальный DOM в React?"
- "Найти максимальную подстроку без повторяющихся символов"

**Плохо:**
- "Расскажи всё про JavaScript" (слишком широко)
- "Помоги" (неконкретно)
- Длинные многосоставные вопросы

### Оптимизация ответов

Ассистент оптимизирован для:
- Технических собеседований на позиции разработчика
- Русскоязычных интервью
- Быстрых, точных ответов

Если ответ слишком длинный или короткий:
- Попробуйте другой режим
- Переформулируйте вопрос более конкретно
- Используйте режим шпаргалки для фактов

## Примеры использования

### Сценарий 1: Теоретические вопросы

```
Режим: Собеседование
Вопрос: "Что такое SOLID принципы?"
Ответ: "SOLID - пять принципов ООП: Single Responsibility (один класс - 
одна ответственность), Open/Closed (открыт для расширения, закрыт для 
модификации), Liskov Substitution (подклассы заменяют базовые классы), 
Interface Segregation (много специфичных интерфейсов лучше одного общего), 
Dependency Inversion (зависимость от абстракций, а не реализаций)."
```

### Сценарий 2: Алгоритмическая задача

```
Режим: Алгоритмы
Вопрос: "Развернуть связный список"
Ответ: "Используй три указателя: prev, current, next. Итерируй по списку, 
меняя направление связей. Сложность: O(n) время, O(1) память.

prev = null
while current:
  next = current.next
  current.next = prev
  prev = current
  current = next"
```

### Сценарий 3: Быстрая справка

```
Режим: Шпаргалка
Вопрос: "HTTP коды ошибок"
Ответ: "200 OK, 201 Created, 400 Bad Request, 401 Unauthorized, 
403 Forbidden, 404 Not Found, 500 Internal Server Error, 502 Bad Gateway, 
503 Service Unavailable"
```

## Ограничения

- Максимум 10 вопросов в контексте (режим собеседования)
- Ответы ограничены 2-5 предложениями
- Оптимизировано для русского языка
- Требуется активное подключение к OpenAI API

## Настройка

Режим по умолчанию можно изменить в настройках приложения:

```typescript
{
  "defaultMode": "interview", // или "general", "algorithm", "cheatsheet"
  "interviewSettings": {
    "saveHistory": true,
    "maxHistoryItems": 10,
    "showConfidence": false
  }
}
```
