## Проверка изменений

## Запуск приложения

### 1. Установка зависимостей (если еще не установлены)

```bash
# В корне проекта
bun install
```

### 2. Запуск API сервера

```bash
# В одном терминале
cd apps/api
bun run dev
```

### 3. Запуск Desktop приложения

```bash
# В другом терминале
cd apps/desktop
bun run tauri:dev
```

## Что тестировать

### 1. Pre-roll буфер (начало речи)
- Начните говорить сразу после запуска
- ✅ Первое слово должно быть захвачено полностью

### 2. Накопление речи
- Задайте длинный вопрос (10-15 секунд)
- ✅ Весь вопрос должен обработаться как единое целое
- ✅ Не должно быть разбиения на части

### 3. Post-roll (конец речи)
- Закончите говорить и подождите 2 секунды
- ✅ Последнее слово должно быть в записи

### 4. Визуализация
- Наблюдайте за Audio Visualizer
- ✅ Плавная анимация без задержек
- ✅ Индикатор состояния (Говорите/Пауза/Слушаю)

### 5. Производительность
- Откройте Task Manager
- ✅ CPU usage должен быть < 2%
- ✅ Memory usage стабильный

## Отладка

### Сохранение аудио для проверки

Нажмите кнопку **"💾 Сохранить аудио"** в приложении.

Файл сохранится в:
```
C:\Users\<username>\Documents\VoiceAssistant\debug\audio_debug_YYYYMMDD_HHMMSS.wav
```

Откройте файл в аудио редакторе и проверьте:
- ✅ Начало речи присутствует
- ✅ Конец речи присутствует
- ✅ Нет обрезанных слов

## Настройки

### Порог голоса

В режиме интервью есть компонент **Voice Sensitivity**.

Можно настроить порог детекции голоса:
- По умолчанию: 2%
- Если много ложных срабатываний: увеличьте до 3-4%
- Если не детектирует речь: уменьшите до 1%

## Известные особенности

### Первый запуск
- При первом запуске может потребоваться время на инициализацию WASAPI
- Это нормально, последующие запуски будут быстрее

### Фоновый шум
- Система фильтрует фоновый шум по порогу
- Если в комнате очень тихо, можно уменьшить порог
- Если шумно, увеличьте порог

### Длинные вопросы
- Максимальная длительность буфера: 30 секунд
- Если вопрос длиннее, старые данные будут удалены
- Это защита от переполнения памяти

## Производительность

### Ожидаемые показатели

| Метрика | Значение |
|---------|----------|
| CPU usage (idle) | < 1% |
| CPU usage (speaking) | < 2% |
| Memory usage | ~50-100 MB |
| Latency (audio level) | ~2ms |
| Latency (processing) | ~2-5 сек |

### Если производительность низкая

1. Проверьте Task Manager - нет ли других процессов
2. Закройте ненужные приложения
3. Проверьте логи на ошибки
