# Анализ производительности и узкие места

## Текущий пайплайн обработки

```
1. Детекция паузы (2 секунды)           ⏱️ 2000ms
2. Post-roll задержка                   ⏱️ 500ms
3. Получение аудио из буфера            ⏱️ ~10ms
4. Сохранение в WAV файл                ⏱️ ~50ms
5. Чтение WAV файла обратно             ⏱️ ~50ms
6. Отправка в Whisper API               ⏱️ 2000-5000ms ⚠️
7. Детекция вопроса (GPT-5-mini)        ⏱️ 500-1000ms ⚠️
8. Генерация ответа (GPT-5)             ⏱️ 2000-5000ms ⚠️
───────────────────────────────────────────────────────
ИТОГО:                                  ⏱️ 7-14 секунд
```

## Узкие места

### 🔴 Критические (API вызовы)

1. **Whisper API** - 2-5 секунд
2. **GPT-5-mini (детекция)** - 0.5-1 секунда
3. **GPT-5 (генерация)** - 2-5 секунд

### 🟡 Средние (можно оптимизировать)

4. **Пауза 2 секунды** - необходима для детекции конца речи
5. **Post-roll 500ms** - необходима для захвата конца
6. **Сохранение/чтение WAV** - ~100ms (избыточно!)

### 🟢 Незначительные

7. **Получение буфера** - ~10ms
8. **Конвертация данных** - ~10ms

## Оптимизации

### 1. ⚡ Убрать сохранение/чтение WAV файла

**Проблема:** Сейчас мы:
1. Сохраняем PCM в WAV файл на диск
2. Читаем WAV файл обратно
3. Отправляем в API

**Решение:** Конвертировать PCM → WAV в памяти

**Экономия:** ~100ms

---

### 2. ⚡ Параллельная обработка

**Проблема:** Сейчас последовательно:
```
Whisper → Детекция вопроса → Генерация ответа
```

**Решение:** Можно начать генерацию раньше

**Варианты:**

#### A. Убрать детекцию вопроса (самое быстрое)
```
Whisper → Генерация ответа
```
**Экономия:** 0.5-1 секунда

**Минусы:** Будет отвечать на все (приветствия, шум и т.д.)

#### B. Детекция вопроса параллельно с подготовкой
```
Whisper → [Детекция вопроса || Подготовка контекста]
       → Генерация ответа
```
**Экономия:** ~200-300ms

---

### 3. ⚡ Streaming ответов

**Проблема:** Ждем полного ответа от GPT-5

**Решение:** Использовать streaming API

```typescript
const stream = await openai.chat.completions.create({
  model: "gpt-5",
  messages: [...],
  stream: true,
});

for await (const chunk of stream) {
  // Показываем ответ по мере генерации
  updateResponse(chunk.choices[0]?.delta?.content);
}
```

**Экономия:** Пользователь видит ответ сразу (воспринимаемая скорость +80%)

---

### 4. ⚡ Уменьшить паузу детекции

**Проблема:** Ждем 2 секунды тишины

**Решение:** Адаптивная пауза

```typescript
// Короткие вопросы - короткая пауза
const SILENCE_DURATION = audioBuffer.length < 100000 
  ? 1000  // 1 секунда для коротких
  : 1500; // 1.5 секунды для длинных
```

**Экономия:** 500-1000ms

---

### 5. ⚡ Кэширование похожих вопросов

**Проблема:** Одинаковые вопросы обрабатываются заново

**Решение:** Кэш ответов

```typescript
const cache = new Map<string, string>();

const cacheKey = transcript.toLowerCase().trim();
if (cache.has(cacheKey)) {
  return cache.get(cacheKey); // Мгновенно!
}
```

**Экономия:** 100% для повторных вопросов

---

### 6. ⚡ Предзагрузка контекста

**Проблема:** Контекст собирается при каждом запросе

**Решение:** Держать контекст готовым

```typescript
// Обновляем контекст заранее
useEffect(() => {
  if (mode === 'interview') {
    preparedContext.current = getInterviewContext();
  }
}, [interviewContext]);
```

**Экономия:** ~10-20ms
